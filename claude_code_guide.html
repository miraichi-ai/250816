🖥️クロードコードのスラッシュコマンドとは

📋 結論

スラッシュコマンド（Slash Commands）とは、クロードコード内で使える特別な命令のことです。コマンドの前に「/」（スラッシュ）を付けることで、クロードに特定の動作をしてもらえます。

🔍 背景：なぜスラッシュコマンドが必要なのか

普通にクロードと会話するだけでも十分ですが、スラッシュコマンドを使うことで以下のメリットがあります：

  • 効率性：決まった動作を素早く実行できる
  • 正確性：意図が明確に伝わり、誤解が減る
  • 専門性：プログラミング特有の操作に特化している

🌟 例え話：レストランでの注文

普通の会話は「何か美味しいものが食べたいです」と言うようなもの。スラッシュコマンドは「ラーメン大盛り、チャーシュー抜き」のように具体的に注文するようなものです。どちらも通じますが、後者の方が確実で早いですよね。

⚡必須スラッシュコマンド詳細解説

/init

機能：プロジェクトの初期分析を行う

用途：新しいプロジェクトの構造を理解したい時

実行例：プロジェクト全体の概要とファイル構成を日本語で説明してくれる

/model

機能：AIモデルの切り替え

用途：処理速度や精度を調整したい時

種類：Sonnet（高速・標準）、Opus（高精度・$100+プラン限定）

/logout

機能：アカウントからログアウト

用途：別のアカウントに切り替えたい時

注意：セッション情報がリセットされる

@ファイル名

機能：特定のファイルを指定して操作

用途：大量のファイルから特定のものを選択したい時

例：@index.html と入力してHTMLファイルについて質問

🎯 各コマンドの具体的な使用例

1

/init の使用例

claude
/init

→ プロジェクト全体の分析結果を日本語で表示

2

/model の使用例

claude
/model

3

@の使用例

claude
@index.html このファイルを分析して

→ 指定したファイルのみを対象に分析

🌏日本語設定の重要性と設定方法

📋 結論

クロードコードはデフォルトで英語で動作しますが、設定ファイルを作成することで日本語での回答を得られるようになります。

🔍 背景：なぜ日本語設定が必要なのか

英語での回答は以下の問題があります：

  • 理解に時間がかかる
  • 細かなニュアンスが伝わりにくい
  • 初心者には敷居が高い
  • 日本語のコメントやドキュメントとの整合性が取れない

🎯 ハイライト：設定ファイルの作成手順

1

ホームディレクトリに移動

cd ~

チルダ（~）は「ホームディレクトリ」を意味する

2

.claudeフォルダの確認・作成

ls -a | grep .claude
mkdir .claude

隠しフォルダ（.で始まる）を作成

3

設定ファイルの作成

cd .claude
code claude.md

VS Codeで設定ファイルを開く

4

日本語設定の記述

Always respond in Japanese

この一行を記述して保存

🌟 例え話：辞書の設定

この設定は、スマートフォンの言語設定のようなものです。一度設定すれば、その後すべてのアプリが日本語で表示されるように、クロードも日本語で回答してくれるようになります。

⚠️ 注意点

/helpなどの一部のシステムコマンドは、設定に関係なく英語で表示される場合があります。これはシステムの仕様です。

/modelコマンドと@ファイル指定を組み合わせることで、効率的なコーディング作業が可能になります。

Sonnet バランス型 ⚡ 高速 📊 高精度 $20+ 日常的なコーディング作業
Opus 最高性能 🐌 やや低速 🎯 最高精度 $100+ 複雑な問題解決・重要なプロジェクト

💡 具体的な使用例

🌟 例え話：車の選択

Sonnetは「普通車」、Opusは「高級車」のようなものです。日常の移動（コーディング）には普通車で十分ですが、特別な場面（複雑なプロジェクト）では高級車の性能が必要になります。
